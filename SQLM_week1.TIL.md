# 1️⃣ 개념 정리

## 01. 여러 가지 과제 

<!-- 본인의 희망 직무와 관련하여 어떤 과제가 있을지 생각해보고, 느낀 바를 적어주세요. -->

1. 빅데이터 환경에서 발생하는 역할 변화

최근 기업들이 빅데이터 분석 기반을 도입하면서
데이터는 한 곳에 저장되고, 이를 활용하기 위해 SQL과 같은 데이터 추출 기술이 필수적으로 요구됨

이 과정에서 분석 담당자와 엔지니어는 기존과 다른 새로운 역할을 요구받게 되며
기술 변화에 따라 업무 방식에도 변화가 발생

대표적으로 Hadoop, Redshift, Aginity, Hue 같은 도구를 사용하여
SQL로 데이터를 추출하는 과정이 데이터 분석의 기본 단계

2. 분석 담당자의 과제

(1) 지표를 스스로 정의해야 함

(2) SQL을 직접 사용해야 함

(3) SQL 없이 분석이 불가능

(4) BI 도구만으로는 한계

(5) 리포트 작성 시간이 증가

(6) 새로운 기술 학습 필요


3. 엔지니어의 과제

빅데이터 환경에서는 엔지니어에게도 새로운 문제가 생긴다.

(1) 분석 요구를 이해하기 어려움

(2) SQL 사용 방식이 다름

(3) 반복적인 요청 발생

(4) 비효율적인 협업 구조

(5) 분석 담당자가 원하는 결과가 바로 나오지 않음

4. 분석 담당자 vs 엔지니어 갈등의 원인

원인

사용하는 기술이 다름

업무 목표가 다름

데이터 이해 수준이 다름


-> 서로 이해 부족, 
커뮤니케이션 문제, 
비효율 발생

5. 해결 방향 — 역할 범위 확장

 분석 담당자 → SQL / 데이터 구조 이해
 엔지니어 → 분석 / 데이터 활용 이해

서로 작업 영역 이해

데이터 구조 공유

SQL 능력 향상


6. 빅데이터 시대에 필요한 능력

- 분석 담당자에게 필요한 것

리포트 생성

- 엔지니어에게 필요한 것

## 02. 시스템

<!-- 교재에서 나열된 도구들의 특징을 정리해주세요. -->
**PostgreSQL**

 오픈소스 관계형 데이터베이스(RDBMS)로 다양한 플랫폼에서 사용할 수 있으며 표준 SQL을 잘 지원

일반적인 데이터베이스처럼 테이블 구조로 데이터를 저장

인덱스를 사용하여 빠른 조회

 WITH 구문, 함수, 확장 기능 등 다양한 SQL 기능을 제공

 데이터 양이 매우 큰 경우에는 단일 서버 구조로 인해 성능에 한계가 발생
->중소규모 데이터 분석에 적합

- 특징

관계형 데이터베이스 구조

표준 SQL 지원

설치 및 사용이 쉬움

대용량 데이터 처리에는 한계


**Apache Hive**

 Hadoop 기반의 분산 데이터 처리 시스템 위에서 동작하는 SQL 인터페이스

대용량 데이터를 처리할 때 디스크 I/O가 병목이 되는 문제를 해결하기 위해
 데이터를 여러 서버에 분산 저장하고 병렬로 처리하는 구조를 사용

Hive는 HDFS 위에 저장된 데이터를 SQL처럼 조회할 수 있도록 만들어진 시스템

 내부적으로는 MapReduce 등의 분산 처리 방식으로 실행

대량의 데이터를 처리할 수 있다는 장점이 있지만 실행 속도가 느리고 실시간 처리에는 적합하지 않음

- 특징

분산 파일 시스템 기반

대용량 데이터 처리 가능

SQL 인터페이스 제공

처리 속도가 느림

배치 처리에 적합


**Amazon Redshift**

AWS에서 제공하는 분산형 데이터베이스로 대용량 데이터를 빠르게 처리하기 위해 설계된 시스템

Hive처럼 분산 처리 구조를 사용하지만 파일 기반이 아니라 데이터베이스 형태로 저장하며 컬럼 기반 저장 방식을 사용하여 조회 속도를 높임

클라우드 기반 서비스이기 때문에 필요에 따라 서버 규모를 확장할 수 있으며 대량 데이터 분석을 빠르게 수행할 수 있음

사용량에 따라 비용이 발생하며 효율적으로 사용하기 위해서는 테이블 설계와 노드 구성에 대한 이해 필요

- 특징

분산형 RDB

컬럼 기반 저장

빠른 조회 속도

클라우드 확장 가능

비용 발생


**Google BigQuery**

구글에서 제공하는 클라우드 기반 빅데이터 분석 시스템

Redshift와 비슷하지만 서버를 직접 관리할 필요가 없고 사용한 데이터 양에 따라 비용이 발생하는 구조

데이터 양이 많을수록 비용이 증가하기 때문에 필요한 데이터만 조회하도록 쿼리를 작성해야함

SQL 표준을 지원하며 매우 큰 데이터를 빠르게 분석할 수 있음

- 특징

서버 관리 불필요

사용량 기반 비용

대용량 처리 가능

SQL 지원

쿼리 비용 관리 필요


**SparkSQL**

Apache Spark 기반의 분산 데이터 처리 시스템에서 SQL을 사용할 수 있도록 만든 기능

Spark는 데이터 처리, 머신러닝, 스트리밍, 그래프 처리 등 다양한 작업을 하나의 환경에서 수행할 수 있는 프레임워크

SQL뿐 아니라 Python, Java, Scala 등 다양한 언어와 함께 사용할 수 있으며 대용량 데이터를 빠르게 처리할 수 있음

여러 단계의 데이터 처리 과정을 하나의 프로그램으로 구성할 수 있다는 장점

- 특징

분산 처리 프레임워크

다양한 언어 지원

빠른 처리 속도

복잡한 분석 가능

개발 난이도 높음

## 03. 데이터 
```
✅ 학습 목차
* 데이터의 종류
* 업무 데이터
* 로그 데이터
* 두 데이터를 사용해서 생성되는 가치
```
**업무 데이터**

업무 데이터는 서비스와 시스템을 운영하기 위해 데이터베이스에 저장되는 데이터

대부분 트랜잭션 처리 과정에서 생성되며 정확성이 매우 중요함

(1) 트랜잭션 데이터

사용자의 행동에 의해 생성되는 데이터

- 예시

구매 기록

로그인 기록

결제 기록

주문 데이터

- 저장 내용

날짜

사용자 ID

상품 ID

수량

가격

**마스터 데이터**

서비스에서 사용하는 기준 정보

- 예시

사용자 정보

상품 정보

카테고리 정보

회원 정보

트랜잭션 데이터에는 ID만 저장되는 경우가 많기 때문에
마스터 데이터와 결합해야 의미 있는 분석이 가능

**로그 데이터**

사용자의 행동이나 시스템 상태를 기록한 데이터

- 예시

페이지 방문 기록

클릭 기록

URL

IP

쿠키

접속 시간

- 특징

(1) 시간 순서로 누적됨

(2) 과거 상태를 그대로 유지

(3) 정확도가 낮을 수 있음

(4) 분석 목적 데이터임
